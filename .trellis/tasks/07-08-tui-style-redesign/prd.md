# TUI 样式重设计（按 `.trellis/images` 三图落地）

## Goal

把当前 SolidJS + OpenTUI 的 `asm tui` 从“能用但视觉简陋”的实现，重设计为 `.trellis/images/{skill,source,fix}.png` 所展示的深色现代终端界面：信息层级清晰、面板边界明确、选中态强、快捷键提示像 keycap，且保持现有功能、交互和核心服务逻辑不变。

## 背景与已确认事实

- 用户提供了三张目标图：
  - `.trellis/images/skill.png`：技能 × 智能体矩阵视图。
  - `.trellis/images/source.png`：来源列表视图。
  - `.trellis/images/fix.png`：健康检查 / 修复视图。
- 当前 TUI 已迁移到 SolidJS + OpenTUI，入口与布局在 `src/tui/App.tsx`，主题在 `src/tui/theme/index.ts`，三视图在 `src/tui/views/*View.tsx`。
- 当前实现已有三 tab、搜索、矩阵、来源、健康检查、底部提示和弹窗；本任务应以视觉样式重构为主，不重新发明业务流程。
- 项目 frontend spec 要求：TUI 使用 OpenTUI `<box>`/`<text>` 与 theme token；不要在组件里硬编码颜色；不要修改 `src/core/**`；写操作仍走 plan → confirm → apply → reload/refresh。
- 最近提交已统一键盘交互为 `space` 切换 / `enter` 确认；本任务默认保留现有键位语义，只改展示方式和提示文案样式。

## 目标视觉需求

### R1 — 全局壳层

- 顶部左侧显示产品名 `agent-skills-mesh`，使用青色强调。
- 顶部右侧显示全局摘要：总项数、错误数、警告数；Doctor 视图可额外体现正常数。
- Tab 栏采用目标图样式：`[1 技能×智能体] [2 来源] [3 健康检查]`，当前 tab 使用蓝色文字、深蓝背景/边框和底部蓝色下划线。
- Tab 下方有横向分隔线，整体背景为深蓝黑色。

### R2 — 通用面板与表格

- 主内容区域使用带边框的大面板，表头与行之间有清晰列分隔。
- 选中行使用深蓝高亮横条，左侧有亮蓝竖条；行号统一为两位数字（`01`、`02`）。
- 表格文本使用低饱和灰蓝色；关键状态使用绿色/黄色/红色/蓝色，但必须保留文字/符号冗余。
- 细节区域使用独立带边框卡片，与表格之间有留白。

### R3 — 技能 × 智能体视图

- 搜索框样式对齐 `skill.png`：带边框、左侧搜索图标/文字，右侧 `/` keycap。
- Matrix 表格保留现有数据语义：行=skill，列=agent，单元格仍为 `[on]` / `[off]` / `[+]` / `[-]` / `[!]` / `—`。
- 选中技能下方展示详情卡：左侧图标块，右侧显示技能名、状态、候选数、描述。
- 底部提示栏使用 keycap 样式：`↑↓ 移动`、`space 切换`、`a 行全装`、`d 删除`、`enter 审查`、`m 智能体`、`/ 搜索`、`? 帮助`。

### R4 — 来源视图

- 来源表格对齐 `source.png`：列为 `id`、`类型`、`启用`、`路径 / 元信息`。
- 每个来源可显示两行信息：第一行路径，第二行 URL / branch 等元信息。
- 选中来源下方详情卡显示来源 ID、类型、状态、本地路径、仓库 URL、默认分支、上次更新等可从现有数据推导的信息；缺失字段不伪造。
- 底部 keycap 提示包括 `a 添加`、`u 更新`、`d 移除`、`e/x 启/停`、`enter 详情`、`ctrl+r 刷新`、`? 帮助`、`1/2/3 切换`。

### R5 — 健康检查视图

- 健康检查表格对齐 `fix.png`：列为序号、状态、类型、信息 / 修复。
- 状态显示图标 + 文案：正常、警告、错误；颜色只作为辅助。
- 详情卡展示当前检查项的状态、类型、信息和修复建议；可修复项要清楚标明修复动作，不可修复项显示无需修复/无自动修复。
- 底部 keycap 提示包括移动、`f 修复`、`F 全修`、`ctrl+r 刷新`、`? 帮助`、`1/2/3 切换`。

### R6 — 响应式与可访问性

- 布局应适配常见终端尺寸；在窄屏时优先截断长文本，不破坏核心导航。
- 所有状态必须可在无颜色情况下理解。
- 中英文双语切换继续有效；新增文案走现有 i18n 体系。

## 非目标 / 约束

- 不改 `src/core/**` 域逻辑。
- 不新增 Web/CSS/DOM 方案。
- 不改变写操作安全模型。
- 不做新的功能范围扩张（例如 command palette、鼠标拖拽、全新排序系统）。
- 不要求像素级完全复刻图片中的每个字符位置；目标是视觉语言、信息结构和交互提示显著对齐。
